Code coverage 
    เครื่องชี้วัด (metric) ที่ใช้ในการบอกว่า โค้ดที่เราเขียนถูกทดสอบมากน้อยแค่ไหน ด้วย unit test หรือ integration test

    ประเภทของ Coverage
        1. Statement Coverage   ทดสอบว่า statement แต่ละบรรทัดถูกเรียกหรือไม่
        2. Branch Coverage	    ทดสอบว่าแต่ละทางเลือก (if/else, switch) ถูกทดสอบไหม
        3. Function Coverage	ฟังก์ชันไหนถูกเรียกจาก test บ้าง
        4. Line Coverage	    คล้าย statement แต่ดูระดับบรรทัด

    ทำไม Coverage ถึงสำคัญ    
        1. ช่วยให้รู้ว่า test ครอบคลุม logic จริงหรือยัง
        2. ลดความเสี่ยงของ bug ใน production
        3. ช่วยทีม QA/Dev ทำงานร่วมกันดีขึ้น
        4. ช่วยวัดคุณภาพของการออกแบบ test


    การตีความ
        1. < 50%	ต่ำมาก มีช่องโหว่เยอะควรเพิ่ม test ด่วน
        2. 50–80%	ปานกลาง ควรปรับปรุงให้ครอบคลุม logic
        3. > 80%	ดีมาก มีความมั่นใจสูง	ยังต้องดูคุณภาพ test ด้วย
