Unit tests
    การทดสอบฟังก์ชันหรือโมดูลที่เล็กที่สุดของโปรแกรม (หรือ "unit") เพื่อให้แน่ใจว่าแต่ละส่วนของโค้ดทำงานได้ถูกต้องตามที่คาดหวัง

Integration tests
        เป็นการทดสอบที่ใช้ตรวจสอบการทำงานร่วมกันของหลายๆ โมดูลหรือฟังก์ชันภายในระบบ โดยมุ่งเน้นการตรวจสอบว่าแต่ละส่วนของระบบทำงานร่วมกันได้ถูกต้องหรือไม่
    เช่น การเชื่อมต่อฐานข้อมูล, การติดต่อกับ API ภายนอก, หรือการทำงานร่วมกันระหว่างหลายๆ ระบบ

End-to-End (E2E) tests
        เป็นการทดสอบที่ตรวจสอบการทำงานของระบบทั้งหมดตั้งแต่ต้นจนจบ (จากมุมมองของผู้ใช้งาน) เพื่อให้แน่ใจว่าแต่ละส่วนของระบบทำงานร่วมกันได้ถูกต้องตามที่คาดหวัง 
    การทดสอบแบบ E2E จะทำการทดสอบจากการส่งคำขอ (request) จนถึงการรับคำตอบ (response) เช่น การทดสอบ API, การทดสอบเว็บแอปพลิเคชันที่เชื่อมต่อกับฐานข้อมูล, หรือการทดสอบกระบวนการทั้งหมดของแอปพลิเคชัน
