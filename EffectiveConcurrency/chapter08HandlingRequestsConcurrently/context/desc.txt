Context
        ตัวกลางสำหรับส่งข้อมูลเกี่ยวกับการทำงานของ goroutines เช่น  สั่งยกเลิก (cancel),ตั้ง timeout,ส่งค่า (เช่น userID, requestID)
    มักใช้ในโปรแกรมที่มีหลาย goroutines ทำงานร่วมกัน และต้องมีการควบคุมให้งานหยุดเมื่อไม่จำเป็นต้องทำต่อ


Cancellation
    ยกเลิก context แบบ manual ถ้ามี goroutines หลายตัวใช้ context เดียวกัน เมื่อเรายกเลิก context  goroutine ทุกตัวจะหยุดพร้อมกัน

Timeout
    คือ context ที่ถูกตั้งเวลาไว้ล่วงหน้า ว่าจะหมดอายุเมื่อถึงเวลานั้น
ใช้สำหรับงานที่ต้องหยุดถ้าใช้เวลานานเกินไป เช่น request ภายนอก, call API, อ่านไฟล์ใหญ่
