- sync
Mutex (Mutual Exclusion Lock)
    ใช้สำหรับ lock ทรัพยากรเพื่อป้องกันไม่ให้หลาย go routine เข้าใช้งานพร้อมกัน
    เหมาะสำหรับ การทำงานที่ต้องการให้ go routine เดียวสามารถอ่านกรือเขียนข้อมูลได้ในเวลาเดียวกัน

RWMutex (Read-Write Mutex)
    เป็น Mutex ที่รองรับการแยก lock สำหรับการอ่าน (Read Lock) และการเขียน  (Write Lock)
    Read Lock : อนุญาตให้หลาย  go routine อ่านข้อมูลพร้อมกัน
    Write Lock : อนุญาตให้เฉพาะ go routine เดียวเขียนข้อมูลในเวลานั้น และต้องไม่มีการอ่านหรือเขียนโดย go routine อื่น
    การใช้งาน เหมาะสำหรับกรณีที่การอ่านเกิดบ่อยกว่าการเขียน

NewCond 
    ใช้สำหรับการจัดการ Condition Variable ซึ่งเป็นระหว่่าง synchronization กับ go routine หลายตัวที่ใช้ทรัพยากรร่วมกัน
    เช่น Queue , Buffer , Shared State
    การใช้งาน : รอจนกว่า Condition จะเป็นจริง, ส่ง signal ไปยัง go routine อื่นเมื่อสถานะของทรัพยากรเปลี่ยน
   
    ฟังก์ชันหลักของ sync.Cond
        - Wait() ใช้รอ go routine
            ก่อนเรียก Wait() ต้องล็อก Mutex ด้วย Lock() และจะปลดล็อกชั่วคราวระหว่างรอ.
            เมื่อใช้ Signal() หรือ Broadcast() จะล็อก Mutex กลับก่อนทำงานต่อ
        - Signal() ใช้เรียก go routine หนึ่งตัวที่กำลังรออยู่
        - Broadcast()  ใช้เรียก go routine ทั้งหมดที่กำลังรออยู่

Once
    ฟังก์ชันจะถูกเรียกใช้เพียงครั้งเดียว ตลอดการทำงานของโปรแกรม
    วิธีใช้งาน
        มี  method หลักคือ Do(f func()) ซึ่งรับฟังก์ชัน f เป็นอาร์กิวเมนต์ และฟังก์ชันนี้จะถูกเรียกใช้เพียง ครั้งเดียว แม้ว่า Do จะถูกเรียกหลายครั้งจาก go routine ต่าง ๆ.
