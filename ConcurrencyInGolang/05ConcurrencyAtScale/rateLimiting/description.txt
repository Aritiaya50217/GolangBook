Rate Limiting 
    การควบคุมอัตราการทำงานของโค้ดหรือการส่งคำขอในโปรแกรมเพื่อป้องกันการใช้งานทรัพยากรเกินขีดจำกัด เช่น การควบคุมจำนวนคำขอที่ส่งไปยัง API ในหนึ่งวินาที หรือจำกัดจำนวนงานที่ทำงานพร้อมกันใน Goroutines

    แนวทางที่ดีสำหรับ Rate Limiting ใน Go
        1. กำหนดอัตราอย่างเหมาะสม: ตั้งค่าอัตรา (Rate) และ Burst ให้เหมาะสมกับระบบ
        2. จัดการ Error อย่างถูกต้อง: ตรวจสอบและจัดการกรณีที่คำขอถูกปฏิเสธ (เช่น ระบบเกินขีดจำกัด)
        3. ใช้ไลบรารี rate: ไลบรารีนี้ช่วยจัดการ Rate Limiting ได้อย่างสะดวกและมีประสิทธิภาพ
        4. รองรับงาน Concurrent: ใช้ Goroutines และ Channel ร่วมกับ Rate Limiter เพื่อประมวลผลคำขอพร้อมกัน
        5. Monitor และ Log: เพิ่มการบันทึกข้อมูลเพื่อวิเคราะห์พฤติกรรมของระบบ
