Error Propagation
    วิธีการส่งต่อข้อผิดพลาด (errors) จากฟังก์ชันหนึ่งไปยังอีกฟังก์ชันหนึ่ง เพื่อให้ฟังก์ชันที่เรียกใช้สามารถจัดการกับข้อผิดพลาดนั้นได้ตามความเหมาะสม แทนที่จะจัดการข้อผิดพลาดในแต่ละฟังก์ชันย่อยทั้งหมด


Error Tracing     
    การติดตามเส้นทางของข้อผิดพลาด เป็นแนวทางที่ช่วยให้คุณสามารถเข้าใจและติดตามได้ว่าเหตุการณ์ข้อผิดพลาดเกิดขึ้นในส่วนใดของโปรแกรม และมีบริบทอะไรที่เกี่ยวข้อง    
    การ Trace ข้อผิดพลาดใน Go สามารถทำได้หลายวิธี เช่น การเพิ่ม Context ให้กับข้อผิดพลาด, การใช้ fmt.Errorf กับ %w, หรือการใช้แพ็คเกจสำหรับการติดตามข้อผิดพลาด เช่น errors และ runtime.

    สรุปแนวทางการทำ Error Tracing
        - เพิ่ม Context ให้ Error: ใช้ %w หรือ errors.Wrap เพื่อบอกว่าข้อผิดพลาดเกิดที่ไหนและทำไม
        - เก็บ Stack Trace: ใช้ runtime/debug.Stack หรือแพ็คเกจเสริมอย่าง pkg/errors
        - ตรวจสอบ Error Chain: ใช้ errors.Is หรือ errors.As เพื่อตรวจสอบประเภทของข้อผิดพลาดที่เกิดขึ้น
        - ทำให้ Debug ง่ายขึ้น: แสดงข้อมูล Stack Trace หรือ Error Chain ในจุดที่เหมาะสม เช่น Log
