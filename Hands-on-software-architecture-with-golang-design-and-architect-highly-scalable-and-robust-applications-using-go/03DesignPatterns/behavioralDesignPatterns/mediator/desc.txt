Mediator

    ใช้วัตถุตัวกลาง (Mediator) เพื่อควบคุมการสื่อสารระหว่างวัตถุหลาย ๆ ตัว ทำให้วัตถุเหล่านั้นไม่ต้องติดต่อกันโดยตรง ลด Coupling (ความพึ่งพากัน)

    โครงสร้างของ Mediator Pattern
        1. Mediator Interface กำหนดวิธีให้ component ต่าง ๆ ใช้สื่อสาร 
        2. ConcreteMediator เก็บ reference ไปยัง components และจัดการการสื่อสารระหว่างกัน
        3. Components สื่อสารกับ Mediator แทนที่จะสื่อสารกันเอง

    ข้อดีของ Mediator Pattern   
        1. ลดความซับซ้อนของการเชื่อมโยงระหว่าง components
        2. เพิ่มความยืดหยุ่นในการเปลี่ยนแปลงพฤติกรรมของระบบ
        3. เพิ่มความสามารถในการแยกโค้ดและทดสอบได้ง่ายขึ้น
