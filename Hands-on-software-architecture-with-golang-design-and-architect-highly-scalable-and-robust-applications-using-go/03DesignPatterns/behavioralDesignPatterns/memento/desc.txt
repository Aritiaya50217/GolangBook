Memento Design Pattern

    เป็นหนึ่งใน Behavioral Design Patterns ซึ่งมีจุดประสงค์เพื่อ เก็บสถานะ (state) ของวัตถุหนึ่งไว้ เพื่อให้สามารถย้อนกลับไปยังสถานะเดิมได้ในภายหลัง

    องค์ประกอบของ Memento Pattern 
        1. Originator – วัตถุที่มี state และสามารถสร้าง/เรียกคืน Memento ได้
        2. Memento – ตัวแทนที่เก็บข้อมูล state (โดยไม่เปิดเผยรายละเอียด)
        3. Caretaker – จัดการกับการเก็บ/ดึง Memento แต่ไม่แก้ไขมัน

    ประโยชน์
        1. แยกความรับผิดชอบของการจัดการ state ออกจากระบบหลัก  
        2. รองรับ Undo/Redo ได้ดี
